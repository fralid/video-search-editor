# Frontend Design

Этот навык направляет создание выразительных, готовых к продакшену интерфейсов. Реализуйте рабочий код с исключительным вниманием к деталям и креативности.

## Технический Стек и Правила

1.  **Core**: Используйте HTML для структуры и Javascript для логики.
2.  **Стилизация (CSS)**: Используйте **Tailwind CSS** как основной инструмент стилизации. Это стандарт для быстрой и системной разработки.
3.  **Web App**: Если нужно сложное веб-приложение, используйте фреймворки типа Vite.
4.  **Создание проектов**: Используйте `npx -y`, проверяйте флаги через `--help`, инициализируйте в текущей директории (`./`), используйте неинтерактивный режим.
5.  **Без плейсхолдеров**: Если нужны изображения, используйте инструмент генерации изображений.

## Эстетика и Дизайн

Избегайте "AI slop" (скучной, шаблонной эстетики). Стремитесь к премиальному качеству.

### Принципы
-   **Смелость**: Выберите четкое направление (минимализм, брутализм, глассморфизм и т.д.) и следуйте ему.
-   **Типографика**: Красивые, уникальные шрифты (Google Fonts: Inter, Roboto, Outfit и др.).
-   **Цвет**: Гармоничные палитры, темные темы, градиенты. Избегайте "дефолтных" цветов.
-   **Динамика**: Hover-эффекты, микро-анимации, плавные переходы. Интерфейс должен ощущаться живым.

## Доступные Ресурсы

### Workflows

-   **Создание нового проекта**: `workflows/new_web_project.md`
    -   Используйте для развертывания (scaffolding) новых веб-приложений с нуля.

## Процесс Разработки

1.  **Поймите задачу**: Уточните требования и контекст.
2.  **Выберите стиль**: Определите эстетическое направление.
3.  **Создайте основу**: `index.css` с переменными и базовыми стилями.
4.  **Компоненты и страницы**: Собирайте интерфейс из переиспользуемых частей.
5.  **Полировка**: Проверьте анимации, отступы и общее впечатление.
