# Формат TaskSpec

TaskSpec — это структурированный документ для передачи задачи между моделями или сессиями.

## Обязательные секции

### 1. Заголовок

```
# Task: [Название]
```

Краткое, понятное название задачи.

### 2. Контекст

```
## Контекст
[1-2 предложения]
```

Почему эта задача важна. Что сломано или чего не хватает.

### 3. Требования

```
## Требования
- [ ] Требование 1
- [ ] Требование 2
```

Конкретные, проверяемые пункты. Используй чекбоксы.

### 4. Файлы для модификации

```
## Файлы для модификации
| Файл | Что сделать |
|------|-------------|
| `path` | Описание |
```

Точные пути к файлам и краткое описание изменений.

### 5. Критерии готовности

```
## Критерии готовности
- [ ] Критерий 1
- [ ] Критерий 2
```

Как проверить, что задача выполнена.

## Опциональные секции

### Ограничения

```
## Ограничения
- Ограничение 1
```

Что нельзя менять или ломать.

### Примечания

```
## Примечания для исполнителя
[Подсказки, контекст, ссылки]
```

Дополнительная информация, которая поможет исполнителю.

### Примеры кода

```
## Примеры
\`\`\`python
# Ожидаемый результат
\`\`\`
```

Конкретные примеры желаемого результата.

## Антипаттерны

❌ **Слишком общие требования**: "Сделай лучше"  
✅ **Конкретные требования**: "Добавь кэширование на 5 минут"

❌ **Отсутствие контекста**: "Исправь баг"  
✅ **С контекстом**: "Баг: при пустом списке падает с NullPointerException"

❌ **Нет критериев**: "Когда будет готово, скажи"  
✅ **С критериями**: "Готово, когда тесты проходят и lint чистый"
